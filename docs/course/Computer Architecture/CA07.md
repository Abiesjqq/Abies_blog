## MIMD概览

并行的趋势: ILP --> TLP --> MIMD. TLP意味着多个程序计数器, 因此通常通过MIMD实现. 

两种MIMD架构: 

- **Multi-processor system**: 系统中只有一个统一的地址空间, 所有处理器共享. 但物理实现上, 可以物理分布、逻辑共享（NUMA）. 
- **Multi-computer system**: 每个处理器都有私有内存（local memory/private memory）, 其他处理器不能直接访问, 而是通过网络发送消息. 通常使用NORMA（no-remote memory access）模型, 即每个节点只能直接访问本地内存, 远程数据需要massage passing. 

## Multi-Processor System

下面所有方法均共享内存, 区别在于处理器访问不同内存位置的时间是否相同, 以及数据是放在普通内存中还是以缓存形式迁移. 

并行处理的挑战在于: 

- 程序中可并行的部分有限: 当有n个处理器时, 理论speedup为n; 但考虑可并行指令占比为k（k<1）, 则实际speedup为

$$
\text{speedup} = \frac{1}{(1-k)+\frac{k}{n}}
$$

- 处理器之间通信有成本: 对于remote memory访问, 访问时该处理器的其他操作暂停, 产生通信开销. 此时, 总CPI为base CPI+远程访问指令的比例乘带来的周期数延迟. 

### UMA

UMA（uniform memory access, 统一内存访问）的物理内存为集中式. 核心特征为一致性访问, 即任何处理器访问任意内存单元的时间相同. 也称为Symmetric Multiprocessor. 

处理器通过ICN与shared memory连接, ICN可以是bus、crossbar或Multistage Network. 每个处理器可以加各自的cache或memory. 

优点为模型简单; 缺点为可扩展性差, 当处理器数量增加时, 总线和内存会变成瓶颈. 

### NUMA

NUMA（non-uniform memory access, 非统一内存访问）的物理内存为分布式. 核心特征为非一致性访问, 即所有处理器共享一个统一地址空间, 但访问本地内存快, 访问远程内存慢（因为访问remote memory需要load/store）. 

每个处理器都有自己的本地内存, 通过ICN连接. 各local memory在物理上分布于不同节点中, 但软件层面为统一地址空间. 

按是否维护cache一致性, 分为两类: 

- NC-NUMA（non cache）: 不自动维护缓存一致性. 如果不同cache中缓存统一数据, 不自动保持一致. 
- CC-NUMA（coherent cache）: 支持缓存一致性. 硬件复杂度高, 但常见. 

### COMA

COMA（cache only memory access）是NUMA的特殊形式. 普通NUMA中, 内存块通常有固定的home node; 但COMA中可以根据使用情况在节点间迁移, 即所有缓存构成统一地址空间. 每个数据初始可以分配在任意位置. 

COMA的优点为, 如果某个数据被某个节点频繁使用, 可以移到该节点附近, 从而降低访问延迟. 但实现复杂, 必须对每个节点维护directory. 

### 缓存一致性问题

多核系统中, 内存中同一数据在不同cache中有多份拷贝. 其他处理器写后, 本地cache中的数据过期, 即cache coherence problem. 

两种一致性问题: 

- Cache coherence: 针对同一地址, 各处理器看到的数据应该一致.
- Memory consistency（cache consistency）: 针对不同地址, 各处理器看到的操作顺序应该满足一定条件.

Cache coherence protocol分为bus snooping protocol和directory based protocol两类. 

#### Snoopy Coherence Protocol

监听一致性协议用于UMA. 所有处理器监听总线上的访问请求; 当某个private cache写数据后, 立刻在总线广播信息; 其他处理器监听到信息后执行对应操作, 如置为invalid、或同步更新本地拷贝、或将修改后数据写回. 

Snoopy协议处理写操作时, 有write invalidate和write update两种策略, 前者更常用. 

下面为几种常见的write invalidate协议. 

- **MSI协议**: 每个block有invalid、shared和modified三种状态, modify下其他拷贝和内存中均无效. 每次读后, modify块的数据写回, 所有拷贝块的状态变为shared; 每次写后, 当前cache块为modified、其他拷贝块为invalid. 
- **MESI协议**: 相比于MSI协议, 增加exclusive状态, 表示某个block只存在于一个cache中、且未被修改. 当某个块第一次被读时, 状态为exclusive; 此后再被其他处理器读, 则都变为shared; 写操作同MSI. 
- **MOESI协议**: 相比于MESI协议, 增加owned状态, 表示当前cache中为写后的脏数据, 其它cache可以有副本. modify状态下其他cache读, 则当前cache变为owned, 将数据发送给读的cache, 其他cache从invalid变为shared. 

MESI和MSI: MSI中, 每次写需要在总线上广播, 以将其他拷贝置为invalid; 而MESI中, E状态下写不需要广播, 对私有数据的读后写更高效. 

MOESI和MESI: MESI中, M状态下其他cache读, 必须写回内存, 再全部为S; 而MOESI中, 允许脏数据被共享, 但指定一个Owner负责维护最新值, 减少写回的开销. 

#### Directory Protocol

目录协议用于NUMA. 相比于Snoopy Coherence Protocol, 用directory代替总线广播invalid信息. 维护一个directory, 记录每个内存块当前被哪些节点缓存、状态是什么、是否被修改. 当某个private cache写后, 在目录中查到哪些节点有副本, 再point-to-point给这些节点发送invalid信息. 

每个cache block的状态与更新同MSI/MESI.

每个directory block有三种状态（和cache block的含义不同）: 

- uncached: 没有任何cache有这个block
- shared: 一个或多个cache有只读的共享副本, 内存是最新的
- exclusive: 只有一个cache有最新副本, 内存过期

更新逻辑: 

- 对shared block和uncached block: 读后变为shared, 写后变为exclusive. 
- 对exclusive block: 目录中记录owner字段. owner自己读写, 则状态不变; 其他节点读, 则数据写回、全部变为shared; 其他节点写, 仍为exclusive, 但owner字段转移; owner自己被替换, 则写回、变为uncached. 

处理器Pi读流程: 

```
if 在local cache中:
    switch cache line状态:
        case shared: 直接返回, 结束.
        case modified: 直接返回, 结束.
        case invalid: 
            同else部分处理;
else:
    向对应处理器Pj发送读请求和读的tag;
    switch Pj directory中tag对应block的状态:
        case shared: 
            Pj directory的block处理器增加Pi;
            Pj将Pj memory中数据返回给Pi;
        case uncached: 
            Pj directory的block状态改为shared, 处理器增加Pi;
            Pj将Pj memory中数据返回给Pi;
        case exclusive:
            Pj向Pj directory中记录的owner处理器Pk发送fetch信号;
            Pk cache将对应block写回, Pk cache line的状态从modified改为shared;
            Pj directory对应block的状态从exclusive改为shared, 处理器增加Pi;
            Pk将最新数据发送给Pj, Pj再把数据返回给Pi;
    Pi cache增加对应line, 状态为shared.
```

处理器Pi写流程: 

```
if 在local cache中:
    switch cache line状态:
        case modified: 
            直接在cache中写, 结束.
        case shared:
            Pi向对应处理器Pj发送写请求和写的tag;
            for 处理器Pk in Pj directory中记录的共享该block的其他处理器:
                Pj向Pk发送invalidate信号;
                Pk cache line状态从shared改为invalid;
            Pj directory对应block的状态从shared改为exclusive, 处理器为Pi;
            Pi cache line的状态改为modified;
            Pi cache中写数据, 结束.
        case invalid: 
            同else部分处理;
else:
    Pi向对应处理器Pj发送写请求和写的tag;
    switch Pj directory中tag对应block的状态:
        case uncached:
            Pj directory对应block的状态改为exclusive, 处理器为Pi;
        case exclusive:
            Pj向Pj directory中记录的owner处理器Pk发送invalidate信号;
            Pk cache line状态从modified改为invalid;
            Pj directory对应block的处理器从Pk改为Pi;
            Pk将最新数据发送给Pj, Pj再把数据返回给Pi;
        case shared:
            for 处理器Pk in Pj directory中记录的共享该block的其他处理器:
                Pj向Pk发送invalidate信号;
                Pk cache line状态从shared改为invalid;
            Pj directory对应block的状态从shared改为exclusive, 处理器为Pi;
    Pj将数据返回给Pi;
    Pi cache增加line, 状态为modified, 数据为新数据;
```


### 内存一致性问题

Memory Consistency问题指, 对各个处理器而言, 看到的不同地址的读写顺序是否一致. 如示例中, Processor 1和2各自都先写再读, 但若都读到对方未写的状态, 则不满足memory consistency. 

```c
// Processor 1
A = 0;
...
A = 1;
if (B == 0) ...

// Processor 2
B = 0;
...
B = 1;
if (A == 0) ...
```

内存一致性模型（Memory Consistency Model）为维护memory consistency的一套规则, 分为Sequential Consistency和Relaxed Consistency Models. 

#### 顺序一致性

要求: 

- 每个处理器内部的内存访问顺序必须保持程序顺序; 
- 不同处理器的访问可以任意交错; 
- 整体执行结果必须等价于某一种全局交错顺序. 

代价为, 现代处理器中使用的cache、乱序等可能让内存操作完成顺序和程序顺序不同. 该方法限制了硬件优化. 

#### 松弛一致性

要求: 允许普通读写乱序完成, 但通过同步操作来强制必要的顺序. 分为: 

1. **Total Store Ordering (TSO)**: 放松RAW. 处理器先把写操作放入store buffer, 然后继续执行后面其他地址的读操作. 这样后面的读可先完成, 而前面的写还没有对其他处理器可见. 
2. **Partial Store Ordering (PSO)**: 进一步放松WAW. 不同写操作之间的顺序也可以变化. 
3. **Weak Ordering (WO)**: 放松几乎所有普通读写之间的顺序, 通过acquire获取锁、release释放锁. release consistency要求, release之前的写, 必须在release之前对外可见; 在acquire之后的读写, 不能被重排到acquire之前. 

## Multi-Computer system

MPP（Massively Parallel Processors, 大规模并行处理机）通常由大量处理器节点组成, 通过高性能互连网络连接. 贵但性能强. 

COW（Cluster of Workstations, 工作站集群）由大量普通PC或工作站通过商用网络连接起来. 便宜但灵活. 

WSC（Warehouse-Scale Computer, 仓库级计算机）为大型互联网公司的数据中心, 面向互联网服务和云计算. 

DSA（Domain-Specific Architecture, 领域专用架构）为针对某一类应用的固定计算模式, 设计专门硬件, 如CNN.
