## SQL中的数据类型

除了内置类型外，还有 用户自定义类型：

- type：将已有的数据类型重命名，e.g.`CREATE TYPE student_name AS VARCHAR(20)`
- domain：将已有的数据类型重命名并可加约束，e.g.`CREATE DOMAIN dollars AS numeric(12, 2) NOT null`

大型数据可用blob（二进制形式）和clob（字符形式）存储。数据库中存储文件的路径；或存储文件的元数据（主要是文件名），在应用程序中配置路径。

## 完整性约束

完整性约束用于保证数据一致性，由DBMS维护，包括：

- NOT NULL
- PRIMARY KEY
- UNIQUE
- CHECK (P)

SQL中，可以写在domain的定义后，也可以指定约束的名称，e.g.：

```sql
CREATE DOMAIN hourly_wage NUMERIC(5, 2)
CONSTRAINT value_test CHECK(value >= 4.00)
```

约束一般不用立即检查，而是在事务结束前检查并处理。否则无法进行一些操作，如插入两个tuple且互为外键。

CHECK约束只拒绝false，可以插入null。

### 参照完整性约束

**UNIQUE**：UNIQUE即指定属性成为candidate key。

UNIQUE的SQL语句：

```sql
-- during initialization
attribute_name VARCHAR(20) UNIQUE

-- after initialization
UNIQUE KEY attribute_name
```

**外键**：在一个关系中作为外键（foreign key）的属性值，必须在另一个关系的主键（primary key）中出现过或为null。被参照关系中，可任意插入；但参照关系中插入和更新时，要是否和被参照关系一致。删除时检查，选择不删除或级联删除。

外键的SQL语句，外键account_number指向account的主键：

```sql
-- during initialization
account_number CHAR(10) REFERENCES account

-- after initialization
FOREIGN KEY (account_number) REFERENCES account
```

定义schema时，可指定级联形式：

- `ON DELETE CASCADE`或`ON UPDATE CASCADE`，表示可级联删除或更新。
- `ON DELETE SET NULL`，表示删除时设为null。认为null值满足外键约束，尽量避免set null。
- `ON DELETE SET DEFAULT`，表示删除时设为该属性的默认值。

级联删除或更新形成级联链条。当级联操作导致约束冲突时，如其中某个属性未设置为级联时，事务abort。

### 断言

Assertion（断言）用于在每次修改后检查数据，当不满足时报错。

创建断言：`CREATE ASSERTION assertion_name CHECK predicate`

当检查“所有...都满足...”时，用`NOT EXISTS`。E.g.：

```sql
CREATE ASSERTION sum_constraint CHECK (
    NOT EXISTS (
        SELECT ... FROM ...
        WHERE (
            ...
        )
    )
)
```

### 触发器

Trigger（触发器）用于在特定数据修改下，自动执行某些操作。由condition和action两部分组成。

创建触发器格式：

```sql
CREATE TRIGGER trigger_name AFTER UPDATE ON table_name
    REFERENCING new row AS nrow
    FOR EACH ROW
        WHEN nrow.attribute_name < 0
            BEGIN atomic
            ...
            END
```

- 需要检查修改后的新值时，用`AFTER`；需要检查旧值时，用`BEFORE`。
- Trigger event可选`UPDATE`, `DELETE`, `INSERT`。当为UPDATE时，可指定列名，格式为`UPDATE OF table_name ON attribute_name`。
- `REFERENCING`表示新旧行的重命名，可以重命名多个值。
- `FOR EACH ROW`表示对每行检查。`FOR EACH STATEMENT`表示对所有行检查。

有时需要用触发器监测外部变化。但由于trigger只能作用于表，需要新建表来表示这种外部关系。

!!! examples "Trigger示例"

    银行存款为负时，设置存款为零、贷款为负数部分。

    ```sql
    CREATE TRIGGER overdraft_trigger
    AFTER UPDATE ON account
    REFERENCING NEW ROW AS nrow
    FOR EACH ROW
    WHEN nrow.balance < 0
    BEGIN ATOMIC
        INSERT INTO loan -- 根据外键约束，要先插入loan
        VALUES (nrow.account_number, nrow.branch_name, -nrow.balance);

        INSERT INTO borrower (
            SELECT customer_name, account_number
            FROM depositor
            WHERE nrow.account_number = depositor.account_number
        );

        UPDATE account
        SET balance = 0
        WHERE account.account_number = nrow.account_number;
    END;
    ```

## 权限管理

### 授权与回收

- 数据方面权限：read，insert，update，delete
- schema方面的权限：index，resources，alteration，drop（没有针对tuple的权限）

授权时可以授予view的权限，而不提供底层表的权限。执行操作时，先检查对视图名的权限，再用视图的定义替换执行。

权限的传递用图表示。DBA（database administrater）有最高权限，边 $A\to B$ 表示A给B授权。图中所有边必须被包含在某条从DBA出发的路径中，必须无回路。上游的权限回收时，下游的权限级联回收。

授权：`GRANT privilege_list ON table_or_view TO user_list`

- privilege_list可以选insert, update, delete, reference等，或`ALL`表示所有权限。
- user_list可以选部分id、public（慎用，后续权限都对所有用户有效）、角色role。
- 可加后缀`with grant option`，表示允许将权限传递给下层用户。

需要给一部分用户授权时，可设置role：

```sql
CREATE ROLE manager;
GRANT SELECT ON table_name TO manager;
```

回收权限：`REVOKE privilege_list ON table_or_list FROM user_list [RESTRICT/CASCADE]`

- user_list中可写`PUBLIC`，表示所有用户失去通过public授权得到的权限，但不用public授权的权限不被回收。
- 如果权限有级联传递，但设置为restrict，则回收权限操作失败。

如果用户被不同上游用户多次授权，则一次回收后，仍保留其他用户授予的权限。

### 审计

审计（audit）用于记录数据库的行动，包括用户、执行的操作、时间等。

语句审计：记录指定用户每次成功（或不成功）执行的对某张表的操作，格式为

```sql
AUDIT table_name BY user_name
BY ACCESS
WHENEVER SUCCESSFUL
```

- `BY user_name`缺省时，默认对所有用户。
- `BY ACCESS`表示对每条语句记录，`BY SESSION`表示每次会话期间，相同类型的语句仅记录一次。
- 取消审计，将AUDIT改为`NOAUDIT`，其余相同。

对象审计：记录所有用户对某张表的特定操作，格式为

```sql
AUDIT operation_name ON table_name
BY ACCESS
WHENEVER SUCCESSFUL
```

审计结果记录在数据字典表`sys.aud$`中，只在DBA下可见。

## 嵌入式SQL

SQL可嵌入其他语言（被称为host language，宿主语言）。下面以嵌入COBOL为例。

用`EXEC SQL`表示嵌入开始，`END_EXEC`表示结束（有分号的语言中，用分号表示结束），中间直接写sql语句。宿主变量前加`:`可代入sql语句中。

定义宿主变量：

```cobol
EXEC SQL begin declare section;
int t;
EXEC SQL end declare section;
```

E.g.，多行查询：先创建游标（cursor），再执行游标，并将结果存入宿主变量中。

```cobol
EXEC SQL
    DECLARE CURSOR c FOR
        SELECT ...
END_EXEC

EXEC SQL OPEN c END_EXEC
EXEC SQL FETCH c INTO :cn END_EXEC
EXEC SQL CLOSE c END_EXEC
```

**Dynamic SQL**指系统能在运行编译、提交sql。通常带有`?`占位符。

JDBC和ODBC分别是Java和其他语言和数据库系统交互的API。
