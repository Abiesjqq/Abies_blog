# 指令级并行（ILP，Instruction-Level Parallelism）

乱序情况下，data hazard中新出现WAR、WAW：

- WAR：实际为先读后写（实际可以写到其他寄存器），乱序后写指令提前到读之前
- WAW：两条写，乱序下先后顺序改变

重命名可一定程度上消除WAR和WAW。

CPU是动态调度。指令乱序执行时，流进流水线和流出顺序不同。

## Scoreboard算法

**_Q：新增什么部件？_**

计分板（scoreboard），功能部件（一个加法，一个除法，两个乘法，一个立即数计算）。

**_Q：流水线有什么调整？_**

将ID段分为IS（issue）、RO（read operands）两段，IS检测structure hazard，决定能不能进流水线；RO检测data hazard，用于实现乱序。

**_Q：Scoreboard上记录什么信息？_**

下面表记录当前在流水线中的指令的信息：

1. 指令状态表：对每条指令，记录各个阶段是否ready。
2. 功能部件状态表：对每个功能部件，记录是否busy，源操作寄存器是什么，源操作数来自的功能部件是什么，结果寄存器是什么。
3. 结果寄存器表：当前在流水线指令，通过功能部分后，结果存放的位置。对每个寄存器，记录是否为某个功能部件的结果。

- IS、RO在上一条的WB的下一拍
- IS阶段不乱序

**_Q：乱序怎么体现？_**

顺序进入流水线，快的指令更早完成EX，乱序流出（WB）流水线。

**_Q：Scoreboard算法有什么问题？_**

1. 检测到name dependence后等待，而不主动解决（如重命名）
2. 未按照程序原来顺序执行，可能导致非精确异常。
